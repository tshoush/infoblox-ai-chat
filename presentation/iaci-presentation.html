The problem

WAPI is powerful — and unforgiving.

📚Steep syntax

Hundreds of object types, strict fields, _ref handling. Lots of docs and curl just to make one change.

⚠️Real blast radius

A typo or duplicate can disrupt DNS/DHCP. Changes hit production network infrastructure directly.

🔒Access vs. safety

You want more people to self-serve — without handing everyone raw API keys to the Grid.

Guardrails

Built for production change-control.

🛡️Pre-flight

⚠️ already exists · 🛑 destructive delete — shown before you run, with the existing object's details.

👥RBAC + approval

Reads open; writes need an operator; deletes need an admin. The approver is re-checked — not just the requester.

🎫Tickets & audit

Optionally require a Jira/ServiceNow ref; single-use approvals (idempotent); every execution logged with who, what, ticket.
